# `MapApi` interface as the cross-feature seam

`components/map-view/api.ts` defines a narrow `MapApi` interface listing every cross-feature operation: `map`, `fitToPhotos`, `reloadGpx`, `forceRemountPopup`, `popupElement`, `markerRadius`, `openExternal`. `<map-view>` `implements MapApi`, with each method a one-line forwarder that does a `renderRoot.querySelector` to the relevant feature element. In-tree consumers (`<map-popup>`, `<map-fit>`, etc.) get `MapApi` via `@consume(mapContext)`; out-of-tree callers (`actions.ts`, panel components) reach it via `document.querySelector('map-view')`. Considered: ad-hoc `document.querySelector` from anywhere, and a global event bus. Both made cross-feature contracts implicit and easy to widen accidentally. With `MapApi` the contract is one file long and adding a cross-feature operation is a deliberate two-step (declare, forward) — `<map-view>` is the only place that knows about its shadow children.
