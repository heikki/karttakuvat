# On-demand image cache, not batch export at sync time

Images are converted from the Apple Photos library on demand: the first request for a given UUID + size triggers an in-process ImageIO/AVFoundation conversion via the native dylib, the result is written to `{dataDir}/cache/{full,thumb}/`, and subsequent requests serve the cached file (validated by source mtime). The earlier design batch-exported every photo at sync time. Rejected because: (1) sync time scaled with library size — minutes for ten thousand photos — even when most photos would never be viewed; (2) the cache had to be invalidated on every Photos.app edit; (3) disk usage was proportional to library size rather than to what the user actually browses. On-demand pushes work to first-view (cheap, single image) and bounds disk use to recently-viewed items.
