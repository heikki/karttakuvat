# Single exclusive interaction-mode signal

Map-input modes (`placement`, `measure`, `route-edit`) live behind one signal in `@common/interaction-mode`. They're mutually exclusive by construction: entering one fires the previous mode's `onExit`. Each owning feature registers itself in `firstUpdated` with `onEnter`/`onExit`/optional `canEnter`; a single edge watcher inside the module dispatches the transitions. Considered: one boolean signal per mode (correct only if every consumer remembers to clear the others, which is fragile), or per-feature local state (forces every feature to know about every other feature's modes). The current shape makes the invariant impossible to violate and keeps Escape-to-exit, mode entry/exit hooks, and cross-cutting effects (e.g. crosshair cursor) routed through one place.
